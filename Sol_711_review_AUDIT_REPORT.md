## 逐提交修复与人工验收

### 1. `3b8c83e Fix task filter accessible labels`

- 问题：任务工作台 7 个筛选控件没有程序化标签；4 个搜索框都被读成“回车搜索”，3 个下拉框没有名称。
- 影响：键盘和屏幕阅读器用户无法确认当前控件筛选什么。
- 人工验收：进入项目工作台，用浏览器可访问性树或 Tab 聚焦，确认可分别识别“方案筛选、测试类别搜索、测试用例搜索、执行人筛选、状态筛选、DTS单号搜索、测试结果搜索”。

### 2. `23ad5c8 Refresh static assets after ordinary fixes`

- 问题：JS/CSS 缓存 24 小时，但普通修复按规则不升版本；浏览器会继续运行旧代码。
- 修复：产品版本仍为 `7.2.5`，静态资源地址增加内容指纹，并启用 ETag 重新验证。
- 人工验收：打开页面源代码，确认 meta 版本为 `7.2.5`，脚本 URL 类似 `v=7.2.5-<12位指纹>`；修改一个前端文件并刷新后，指纹应改变。

### 3. `0b86e06 Label sample pool search filters`

- 问题：样机列表 5 个下拉筛选在辅助技术中均为无名组合框。
- 人工验收：进入任一样机池，确认搜索框及“使用状态、故障状态、重组状态、挂账人、持有人”筛选均有独立可访问名称。

### 4. `7b0d95c Add keyboard access to sample cards`

- 问题：样机卡片只能鼠标点击，无法用键盘打开详情。
- 人工验收：在样机列表按 Tab 聚焦样机编号，按 Enter，应打开对应样机详情；“档案销毁”仍是独立按钮。

### 5. `d0a2f6a Search sample task history from the pool`

- 问题：界面承诺“履历搜索”，但服务端只搜索样机字段和问题记录，不搜索关联任务与样机事件。
- 证据：真实数据中“低温标准弯折”修复前返回 0 台，修复后返回 26 台。
- 人工验收：在 `Sagitarriasd` 搜索“低温标准弯折”，确认返回 26 台；打开任一结果的测试履历，确认存在对应任务。

### 6. `63d7293 Preserve sample fields in CSV round trips`

- 问题：样机 CSV 导出后再导入会静默丢失带入时间、借用时间、型号、配置及 SKU/版本。
- 修复：导出表头与解析别名闭环；兼容旧模板的单一“方案”列。
- 人工验收：只在临时数据副本中操作。导出一台包含上述字段的样机，再导入空样机池，逐字段比对。

### 7. `7c7e9f0 Prevent narrow-screen layout collapse`

- 问题：400px 视口下侧栏仍占 212px，主区仅 188px，面包屑和分页文字竖排，并出现横向滚动。
- 修复：640px 以下使用 66px 图标侧栏，缩小内容留白并允许分页合理换行。
- 人工验收：将浏览器宽度设为 400px；确认主区约 334px、页面无横向溢出、上一页/下一页不再逐字竖排。

### 8. `175ad5f Protect filtered reassembly pages from stale patches`

- 问题：“重组/非重组”未被识别为有效筛选。样机变更后可能把已不符合筛选条件的样机继续留在当前缓存页。
- 人工验收：只在临时数据副本中操作。启用“重组”筛选，将可见样机改成非重组；保存后该样机应从结果中消失，统计同步更新。

### 9. `f0ebd77 Report both invalid sample person fields`

- 问题：导入错误计数包含挂账人和持有人，但提示把全部错误都说成挂账人错误。
- 人工验收：只在临时导入文件中测试。分别提供非法挂账人和非法持有人，提示应明确为“挂账人或持有人字段”。

### 10. `ab8aedc Restore the HTTP benchmark server surface`

- 问题：Server 重构后不再暴露 `ThreadingHTTPServer`，仓库自带的真实 HTTP 并发基准无法启动。
- 人工验收：运行 `python dev\tools\http_concurrency_benchmark.py`，应完成分页、照片、导入和导出四组基准且错误数为 0。

### 11. `bdb9b87 Enable keyboard navigation for custom controls`

- 问题：侧栏主入口、子入口、折叠器和数据工具均为鼠标专用 `div/span`；已有 `role=button` 控件也不能用 Enter/Space 激活。
- 人工验收：仅用 Tab、Enter、Space 导航首页、项目管理、样机档案池、子目录和导入导出入口；聚焦时应显示蓝色轮廓。

### 12. `920bbed Unify sample analysis and retired colors`

- 问题：“取走分析”在样机池统计中为深红，在样机卡片中却为未知状态灰；已退库也未使用专用状态变量。
- 人工验收：进入样机池与样机列表对照同一状态；“取走分析”圆点、卡片色条和徽章应统一为红色语义，“已退库”使用专用灰色。
